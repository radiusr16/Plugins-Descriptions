Версия 1.5.0 (28 июн 2026 г.):

Добавлено:

  • в команду cks_add_sound добавлена поддержка новых групп звуков:
    suicide - игрок убил себя сам
    knife, p228, scout, xm1014, mac10, aug, elites, fiveseven, ump45, sg550,
    galil, famas, usp, glock, awp, mp5, m249, m3, m4a1, tmp, g3sg1, deagle,
    sg552, ak47, p90 - убит из определенного оружия
    Если игрок был убит объектом, который не вошел в список, то можно указать его
    classname в качестве идентификатора
  • добавлено меню настроек, в котором можно настроить слышимость звуков и
    видимость HUD сообщений
  • добавлен квар cks_settings_default_state, отвечающий за настройки по умолчанию
    в виде флагов, которые определяют слышимость звуков и видимость HUD сообщений
  • добавлен квар cks_menus_numbers_color, задающий цвет цифр пунктов меню
  • добавлена native функция: 
          /**
        * Отображает меню настроек
        *
        * @ param player     индекс игрока
        *
        * @ return           статус операции
        *                    0 - меню не может быть показано
        *                    1 - меню успешно показано
      */
      native cks_menu_display(const player);
  • добавлены контекстно-зависимые аргументы для HUD сообщений:
    [killer] - имя игрока, который сделал фраг
    [victim] - имя жертвы
    Также в lang файл добавлена памятка о них
  • добавлен квар cks_enabled, который динамически включает и отключает плагин.
    Может пригодится для отключения на время разминки через плагин Cvar On Map
  • добавлен квар cks_start_round, который определяет с какого раунда будет работать
    плагин
  • добавлены женские варианты звуков в ‘sound/cso_kill_sounds/female/’

Изменено:

  • квар cks_play_sounds_access_flags переименован в cks_access_flags
  • квар cks_toggle_play_sounds_cmd переименован в cks_settings_menu_cmd
  • текст HUD сообщений заменен на ключи словаря в cso_kill_sounds.cfg
  • в cks_settings_menu_cmd добавлено доп значение: ksound
  • в команде cks_add_sound аргумент sound_path теперь может быть пустым
  • количество звуков в группах теперь не ограничено
  • в HUD сообщения добавлена поддержка переноса строки ^n

Исправлено:

  • постоянно сбрасывался счетчик убийств при cks_kills_count_reset_delay 0

Удалено:

  • удален квар cks_play_sounds_default_state

Обновившиеся файлы:

  • addons/amxmodx/plugins/cso_kill_sounds.amxx
  • addons/amxmodx/configs/cso_kill_sounds.cfg
  • addons/amxmodx/data/lang/cso_kill_sounds.txt

Версия 1.0.7 (19 окт 2017 г.):

Добавлено:

  • добавлена native функция cks_toggle_play_sounds(id), которая позволяет
    встраивать функцию включения / отключения проигрывания звуков при убийстве в
    другие плагины
  • добавлен квар cks_play_sounds_access_flags, отвечающий за флаги доступа для
    игроков, которым будут проигрываться звуки при убийстве
  • добавлен квар cks_toggle_play_sounds_cmd, отвечающий за флаги список команд
    (разделяются ‘;’) с помощью которых будет включаться / отключаться проигрывание
    звуков (‘impulse 100’ - фонарь, ‘impulse 201’ - спрей, если используется say,
    то say_team добавляется автоматически)
  • добавлена команда чата /kill_sounds, с помощью которой каждый игрок может
    включить/отключить проигрывание звуков убийства с сохранением результата
    при выходе с сервера
  • добавлен квар cks_play_sounds_default_state, который определяет будут
    проигрываться (1) или нет (0) звуки, по умолчанию если игрок не менял значение
  • добавлена серверная консольная команда add_kill_sound, которая позволяет
    добавлять звуки (mp3 или wav) в sound list плагина (см. полное описание)

Изменено:

  • удален код блокирующий запуск плагина на ReHLDS серверах, но не проверялась
    работоспособность
